3.1 Ejemplos de uso volver a índice

Con la finalidad de facilitar la comprensión del uso de algunas clases y propiedades de la ontología se proporcionarán a continuación algunos ejemplos.

En primer lugar representaremos un Municipio, como se puede observar en el código de ejemplo, se crea una instancia de la clase esadm:Municipio que, como se explicó en la Figura 1, es una clase que se reutiliza de la ontología de territorio. El nombre del Municipio, su código de identificación y la fecha en que se creó en el catálogo se representan a través de las propiedades de datos dc:title, dc:identifier y dc:created respectivamente. La codificación oficial asociada a la ciudad se representa mediante la propiedad esadm:codigoINE y su geometría se representa mediante la propiedad geosparql:hasGeometry. 

	<https://datos.ign.es/recurso/municipio/28166> a esadm:Municipio ;
	dc:title 		"Valverde de Alcalá" ;
	dc:identifier		"28166" ;
	dc:created		"2012-09-19"^^xsd:date ;
	esadm:codigoINE		"28166" ;
	geosparql:hasGeometry	<https://datos.ign.es/recurso/municipio/28166/geometry> .

Para la representación de la geometría del municipio se crea una instancia de sf:MultiPolygon que es una de las clases definidas dentro de la ontología GeoSPARQL utilizada en la ontología de Objetos Geográficos geo_core. Adicionalmente se emplea la propiedad geosparql:asWKT para representar el tipo de codificación de la geometría; finalmente el tipo de dato del valor de la geometría se representa mediante la propiedad geosparql:wktLiteral. 

	<https://datos.ign.es/recurso/municipio/28166/geometry> a sf:MultiPolygon ;
        geosparql:asWKT  "MULTIPOLYGON (((-5.418825249998759 35.91208060445847, ..., 35.8904970644321)))"^^geosparql:wktLiteral .

La representación de las geometrías de todos los objetos geográficos del catálogo se realizará de forma parecida al ejemplo anterior. Se emplearán las clases, propiedades y propiedades de datos de la ontología GeoSPARQL para representar las líneas, polígonos, etc. requeridos.

Otro ejemplo que usaremos es el de la representación de una Autovía, en este caso la M-30 de Madrid. Al igual que en el ejemplo anterior empezamos con la creación de una instancia de la Autovía, para ello se emplea la clase btn100:Autovia. Al igual que el caso anterior el nombre de la Autovía, su código de identificación y la fecha en que se creó en el catálogo se representan a través de las propiedades de datos dc:title, dc:identifier y dc:created respectivamente. Para representar el tipo de calzada de la Autovía se utiliza la propiedad btn100:calzada que en el caso de la M-30 tiene el valor doble, es decir esta Autovía posee dos calzadas separadas y paralelas con sentidos distintos; sin embargo, pueden existir Autovías cuyo tipo de calzada sea sentido único. Para la representación del valor de la propiedad btn100:calzada se hace uso de uno de los esquemas de conceptos que se crearon para esta ontología y que se detallaron anteriorme en la Figura 8. Para las demás propiedades que se utilizan para la representación de los datos de la Autovía M-30 se procede de forma similar que para el caso del tipo de calzada, como se puede observar en el código para cada propiedad se utiliza el esquema de conceptos asociado. 

	<https://datos.ign.es/recurso/autovia/244> a btn100:Autovia ;
	dc:created               "2013-07-01"^^xsd:date ;
	dc:identifier            "244" ;
	dc:title                 "M-30" ;
	geosparql:hasGeometry    <https://datos.ign.es/recurso/autovia/244/geometry> ;
	btn100:calzada           <https://datos.ign.es/kos/transportes/calzada/doble> ;
	btn100:competencia       <https://datos.ign.es/kos/transportes/competencia/local> ;
	btn100:estadoDeUso       <https://datos.ign.es/kos/transportes/estado-de-uso/en-uso> ;
	btn100:rutaTENT-T        <https://datos.ign.es/kos/transportes/ruta-ten-t/ruta-no-transeuropea> ;
	btn100:situacionDeVia    <https://datos.ign.es/kos/transportes/situacion-via/elevado> ;
	btn100:tipoDeItinerario  <https://datos.ign.es/kos/transportes/tipo-itinerario/no-itinerario-europeo> ;
	btn100:tipoDeTramo       <https://datos.ign.es/kos/transportes/tipo-tramo/interna> .
